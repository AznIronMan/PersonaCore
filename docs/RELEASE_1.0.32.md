# Release 1.0.32

`1.0.32` adds a reusable worker/process operations surface for consumer
consoles that need shared readiness cards, schedule/run telemetry, dead-letter
queues, rollback candidates, dry-run candidates, process feeds, and
review-first control slots while keeping execution and mutations local.

## Added

- `WorkerOperationsSurfaceConfig`, `WorkerReadinessRow`,
  `WorkerScheduleRow`, `WorkerRunTelemetryRow`, `WorkerDeadLetterRow`,
  `WorkerRollbackCandidate`, `WorkerDryRunCandidate`, `WorkerProcessEvent`,
  and `WorkerControlActionSlot`.
- `WORKER_OPERATIONS_FEATURE`, `render_worker_operations_surface(...)`, and
  `worker_operations_surface_feature_enabled(...)`.
- Compact readiness cards, metrics, status tabs, process feed rows, responsive
  tables for schedules/runs/dead letters/rollback/dry-run candidates, and
  trusted action slots for consumer-owned review/control forms.
- Owner-private safe-alternate rendering for run output/errors, dead-letter
  reasons, dry-run summaries, process events, raw links, and row actions.
- Public exports, compatibility shim submodules, doctor checks, fixture
  coverage, visual-smoke coverage, docs, and focused tests.

## Boundaries

PersonaConsole only renders normalized worker/process metadata supplied by a
consumer. Consumers still own worker execution, scheduling decisions, retries,
dead-letter mutations, rollback proposal creation, dry-run producer logic,
database writes, auth, service managers, deployment, and runtime-specific
business rules.

Action-slot HTML is a trusted extension point for consumer-owned controls. It
should be generated by the consuming runtime after its own authorization checks.
