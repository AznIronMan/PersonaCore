# Release 1.0.30

`1.0.30` adds a reusable detail/dossier surface for consumer consoles that need
shared entity profile, record detail, and source-review pages while keeping
runtime data, routes, permissions, forms, and mutations local.

## Added

- `DetailDossierSurfaceConfig` plus header, field, metric, section, source
  table, timeline, related-link, audit-row, and action-slot models.
- `render_detail_dossier_surface(...)` and
  `detail_dossier_surface_feature_enabled(...)`.
- Dense entity-detail layout with avatar/initial fallback, badges, metadata
  fields, metrics, narrative sections, source tables, related links, timeline,
  audit rows, responsive behavior, and runtime-owned action slots.
- Owner-private safe-alternate rendering for fields, sections, table cells,
  timeline entries, and audit rows, with raw private hrefs suppressed unless
  the viewer can see the matching scope.
- Public exports, compatibility shims, doctor checks, fixture coverage, docs,
  and focused tests for escaping, feature gates, empty state, and privacy.

## Boundaries

PersonaConsole only renders normalized record detail data supplied by a
consumer. Consumers still own lookups, auth, route authorization, database
queries, form handlers, mutations, audit logging, deployment, and
runtime-specific business logic.

`body_html` and action-slot HTML are trusted extension points for
consumer-owned controls. They should be generated by the consuming runtime after
its own authorization checks.
