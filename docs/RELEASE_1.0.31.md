# Release 1.0.31

`1.0.31` adds a reusable media/artifact library surface for consumer consoles
that need shared asset galleries, received-media queues, generated-artifact
review, reference anchors, and upload/import action slots while keeping storage
and policy local.

## Added

- `MediaLibrarySurfaceConfig`, `MediaLibraryItem`,
  `MediaLibraryMetadata`, and `MediaLibraryActionSlot`.
- `MEDIA_LIBRARY_FEATURE`, `render_media_library_surface(...)`, and
  `media_library_surface_feature_enabled(...)`.
- Grid and list layouts with stable previews, image/video/audio/file fallbacks,
  preview dialogs, status tabs, filters, metrics, metadata chips, source labels,
  safety/sendability/shareability flags, and runtime-owned action slots.
- Owner-private safe-alternate rendering for titles, details, metadata, and raw
  preview/detail URLs.
- Public exports, compatibility shims, doctor checks, fixture coverage, docs,
  and focused tests for escaping, feature gates, safe URLs, empty state, and
  privacy.

## Boundaries

PersonaConsole only renders normalized media metadata supplied by a consumer.
Consumers still own file storage, uploads, upload validation, byte serving,
provider generation, moderation decisions, route authorization, mutations,
retention, deployment, and runtime-specific business logic.

Action-slot HTML is a trusted extension point for consumer-owned controls. It
should be generated by the consuming runtime after its own authorization checks.
